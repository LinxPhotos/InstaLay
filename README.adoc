= InstaLay

Batch-frame photos for Instagram without idiotic forced crops.

Potential alternate name: `LayMaker`.

* Owner: link:https://github.com/LinxPhotos[Linx] (`LinxPhotos` on GitHub)
* Platforms: Android, iOS, macOS, Linux, Windows, web
* Engine: Flutter
* Repo: https://github.com/LinxPhotos/insta-lay

== What it does

InstaLay puts each photo on a canvas at a ratio you choose (default Instagram feed *4:5*), with a pixel-accurate border and a photographic matte. Thumbnails show the canvas applied. Export with Lanczos-2 / Lanczos-3 (or cubic / linear / nearest). Build SCRL-style tapestry panoramas that slice into carousel frames. Save projects with version history — posting to Instagram freezes a version; clone to keep editing.

== Core features

* Aspect presets: 4:5, 1:1, 16:9, 9:16, 3:2, 2:3, 21:9
* Border thickness in pixels
* Matte swatches grouped by Adams photographic zones (sheer greys/whites, taupes, cinematic blacks) — default *Sheer Cloud*
* Paper textures: fine grain, matte fiber, cold press, baryta
* Batch mode or tapestry/panorama mode
* 1:1 preview sidebar
* Canvas templates for consistent results across photo sets
* Device-local projects with version clone / repost

== Installers and releases

CI builds desktop packages on every `v*` tag:

* Windows x64 / arm64 — ZIP portable + Inno Setup EXE + **Microsoft Store MSIX** (`*-store.msix`)
* macOS arm64 / x64 — DMG + ZIP of `InstaLay.app`
* Linux x64 / arm64 — `.tar.gz` (AppImage when `appimagetool` is present)

Workflows:

* `.github/workflows/release.yml` — multi-arch build + GitHub Release table
